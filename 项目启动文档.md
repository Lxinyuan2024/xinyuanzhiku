## 💰 支付系统设计

### 微信支付方案
```
1. 用户点击订阅 → 生成订单
2. 服务器调用微信支付API
3. 返回微信收款码/小程序支付链接
4. 用户扫码支付
5. 微信回调通知 → 更新用户会员状态
```

### 支付宝方案
```
1. 用户点击订阅 → 创建支付宝订单
2. 生成支付宝收款码
3. 用户扫付款码支付
4. 支付宝回调 → 开通会员权限
```

### 免接口方案（初期建议）
```
使用个人收款码 + 手动开通
1. 用户支付后截图
2. 用户上传截图 + 账号信息
3. 手动在后台为用户开通权限
4. 发送确认邮件/通知
```

## 📱 响应式设计策略

### 设备适配
- **手机端**：375px-767px，单栏布局，大按钮，简洁导航
- **平板端**：768px-1023px，双栏布局，适应阅读
- **桌面端**：1024px+，三栏布局，完整功能展示

### 关键断点
```css
/* 手机端：小于768px */
@media (max-width: 767px) { ... }

/* 平板端：768px-1023px */
@media (min-width: 768px) and (max-width: 1023px) { ... }

/* 桌面端：大于1024px */
@media (min-width: 1024px) { ... }
```
